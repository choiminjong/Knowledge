# v0.1.0

최초 릴리즈 (2026-02-23)

## 주요 기능

### 데이터 수집 (STEP 1)
- 네이버 뉴스 6개 카테고리 자동 크롤링
- Selenium + undetected-chromedriver 기반
- 카테고리: 정치, 경제, 사회, 생활/문화, IT/과학, 세계
- Excel 파일 출력 (`Articles_YYYYMMDD_HHMMSS.xlsx`)

### 그래프 구축 (STEP 2)
- Recursive Chunking 기반 텍스트 분할 (chunk_size=500, overlap=100)
- Neo4j 그래프 스키마: Article, Content, Category, Media, Author
- 관계: HAS_CHUNK, BELONGS_TO, PUBLISHED, WROTE

### GraphRAG 검색 (STEP 3)
- 3종 Retriever: Vector, VectorCypher, Text2Cypher
- ToolsRetriever: LLM이 질문에 따라 적절한 도구 자동 선택
- bona/bge-m3-korean 임베딩 (1024차원)

### 웹 시각화
- FastAPI + vis-network.js 기반 실시간 그래프 시각화
- 2가지 답변 모드: 요약 브리핑 / 기사 목록
- 답변 개수 조절 (k=1/3/5/10)
- 검색 결과 서브그래프 하이라이트
- 노드 클릭 시 상세 속성 패널
- Cypher 쿼리 표시, 소요시간 표시
- 사용된 Retriever 정보 표시

## LLM 설정

| 항목 | 값 |
|------|-----|
| LLM | Ollama `qwen3:8b-q4_K_M` |
| Embedding | Ollama `bona/bge-m3-korean` |
| temperature | 0.1 |
| num_predict | 4096 |
| num_ctx | 8192 |
| repeat_penalty | 1.2 |
| think mode | 비활성화 |

## 프로젝트 구성

- `pyproject.toml` + `uv.lock` 기반 의존성 관리
- `uv sync`로 원클릭 환경 구성
- Python 3.12+
- notebooks/ 폴더로 Jupyter 노트북 정리
- docs/guides/ 폴더에 상세 문서 6개

## 변경 이력

- EXAONE 3.5 모델에서 Qwen3 8B 모델로 전환 (Tool Calling 안정성 향상)
- Qwen3 thinking 모드 비활성화 (속도 개선)
- repeat_penalty 튜닝 (1.5 → 1.2, 답변 깨짐 방지)
- 프롬프트 출력 형식 지정 (일관된 답변 포맷)
- 그래프 하이라이트 로직 개선 (답변 기반 노드 추출)
- Cypher 쿼리 캡처 (ToolsRetriever 메타데이터 유실 우회)

## 알려진 이슈

- 로컬 LLM 특성상 응답 시간이 30초~3분 소요 (GPU 성능에 따라 차이)
- repeat_penalty가 낮으면 동일 기사가 중복 출력될 수 있음
- Text2Cypher가 잘못된 Cypher를 생성할 경우 검색 실패 가능
